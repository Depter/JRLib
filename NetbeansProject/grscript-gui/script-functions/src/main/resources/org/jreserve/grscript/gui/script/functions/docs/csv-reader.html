Import Csv-Data

GRscript provides some convenience methods to read and import csv files as matrices.

  • readCsv(String): reads the file, denoted by the supplied path, using the default settings (see Default Settings).
  • readCsv(File): reads the file, using the default settings (see Default Settings).
  • readCsv(Reader): reads the data from the given reader, using the default settings (see Default Settings).
  • readCsv(String, Map): reads the file, denoted by the supplied path, using the give settings (see Default Settings).
  • readCsv(File, Map): reads the file, using the give settings (see Default Settings).
  • readCsv(Reader, Map): reads the data from the given reader, using the give settings (see Default Settings).
  • readCsv(String, Closure): reads the file, denoted by the supplied path, using the give settings (see Default Settings).
  • readCsv(File, Closure): reads the file, using the give settings (see Default Settings).
  • readCsv(Reader, Closure): reads the data from the given reader, using the give settings (see Default Settings).

Default Settings

  • Decimal Separator: the property denotes the character used as decimal separator. The default value is choosen according to the System Locale. Use [decimal | decimals | decimalseparator] in the map settings, or decimalSeparator() in the builder syntax to change this property.
  • Column Separator: the property denotes the character used as column separator. The default value is the tab character ("\t"). Use [column | columns | columnseparator] in the map settings, or columnSeparator() in the builder syntax to change this property.
  • Row Headers: the property denotes whether the input file contains row titles. The default value is false. Use [rowheader | rowheaders | hasrowheader | hasrowheaders] in the map settings, or rowHeader() in the builder syntax to change this property.
  • Column Headers: the property denotes whether the input file contains column titles. The default value is false. Use [columnheader | columnheaders | hascolumnheader | hascolumnheaders] in the map settings, or columnHeader() in the builder syntax to change this property.
  • Quotes: the property denotes whether the cells in the input files are surrounded with quotes. The default value is false. Use [quote | quotes | hasquote | hasquotes] in the map settings, or quotes() in the builder syntax to change this property.

Examples

String path = "D:/Temp/test.csv"
data = readCsv(path)

data = readCsv(path, [decimal:'.', column:';'])

data = readCsv(path) {
    decimalSeparator '.'
    columnSeparator ';'
    rowHeader true
    columnHeader true
    quotes false
}
