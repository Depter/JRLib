Export Csv-Data

GRscript provides some convenience methods to export data to csv files. In the following methods the data parameter is a one or two dimensional double array.

  • writeCsv(String, data): writes the data to a csv file, denoted by the supplied path, using the default settings (see Default Settings).
  • writeCsv(File, data): writes the data to the file, using the default settings (see Default Settings).
  • writeCsv(Writer, data): writes the data to the given writer, using the default settings (see Default Settings).
  • writeCsv(String, data, Map): writes the data to a csv file, denoted by the supplied path, using the give settings (see Default Settings).
  • writeCsv(File, data, Map): writes the data to the file, using the given settings (see Default Settings).
  • writeCsv(Writer, data, Map): writes the data to the given writer, using the given settings (see Default Settings).
  • writeCsv(String, data, Closure): writes the data to a csv file, denoted by the supplied path, using the given settings (see Default Settings).
  • writeCsv(File, data, Map): writes the data to the file, using the given settings (see Default Settings).
  • writeCsv(Writer, data, Map): writes the data to the given writer, using the given settings (see Default Settings).

Default Settings

  • Decimal Separator: the property denotes the character used as decimal separator. The default value is choosen according to the System Locale. Use [decimal | decimals | decimalseparator] in the map settings, or decimalSeparator() in the builder syntax to change this property.
  • Column Separator: the property denotes the character used as column separator. The default value is the tab character ("\t"). Use [column | columns | columnseparator] in the map settings, or columnSeparator() in the builder syntax to change this property.
  • Row Names: a list, containint the row titles. Use [rownames | row names | row-names] in the map settings, or rowNames(List) in the builder syntax to change this property.
  • Column Names: a list, containint the column titles. Use [columnnames | column names | column-names] in the map settings, or columnNames(List) in the builder syntax to change this property.
  • Header Quots: the property denotes whether the row and column names should be surrounded with quots ("). The default value is false. Use [quotheader | quot header | quot-header] in the map settings, or quotHeader() in the builder syntax to change this property.
  • Value Quots: the property denotes whether the values should be surrounded with quots ("). The default value is false. Use [quotvalue | quot value | quot-value] in the map settings, or quotValues() in the builder syntax to change this property.

Examples

String path = "D:/Temp/test.csv"

//write APC-PAID data, using the default values
writeCsv(path, apcPaid())

//write APC burning cost data, using the default values
writeCsv(path, apcExposure())

//Use map syntax to modify some default values
writeCsv(path, apcPaid(), [decimal:'.', column:','])

//Builder syntax
writeCsv(path, apcPaid()) {
    decimalSeparator '.'
    columnSeparator ';'
    rowNames 1997..2004
    columnNames 1..8
    quotHeader true
    quotValues false
}
